19.3 Stream API

Yuwl 2024-08-18 17:05:44
Categories： Java Tags： lambda

现在我们来学习 Stream API。

Stream API 是 Java 8 引入的功能，用于以函数式风格处理集合数据。它提供了一个强大而灵活的方式来进行数据处理，支持各种操作，如过滤、映射、排序和收集等。以下是对 Stream API 更详细的学习内容。

基本使用

1. 基本概念

Stream 是一个数据的序列，支持各种计算操作，比如筛选、映射、归约等。它有以下特点：

不存储数据：Stream 只是对数据的操作，不会保存数据。
惰性求值：Stream 的操作是惰性执行的，只有在需要结果时才会真正计算。
链式操作：可以将多个操作链式连接起来，形成流水线。

3. 常见操作

  • 中间操作（惰性操作）： 返回一个新的 Stream，原 Stream 不变。

  • filter：根据条件过滤数据。

  • map：对每个元素应用函数。
  • flatMap：将每个元素转换为多个元素并扁平化。
  • sorted：排序。
  • peek：用于调试，可以查看每个元素。

  • 终结操作（执行计算）： 产生结果或副作用，并结束 Stream 操作链。

  • collect：将结果收集到集合中。

  • count：计算元素数量。
  • reduce：将所有元素合并为一个值。
  • forEach：对每个元素执行操作。
  • findFirst：获取第一个元素（如果存在）。

anyMatch、allMatch、noneMatch：检查是否存在满足条件的元素。

5. 性能考虑

  • Stream 的并行操作可以利用多核处理器来提高性能，但需要注意并行流的开销，特别是在数据量小或操作简单时。
  • 惰性求值：Stream 的操作是惰性求值的，即只有在需要结果时才会进行计算。
