6.1 注解

Yuwl 2024-08-04 16:16:52
Categories： Java Tags： java

现在我们来学习 Java 中的注解。

注解（Annotations）是Java提供的一种元数据机制，用于为代码元素（如类、方法、字段等）添加额外的信息。这些信息可以在编译时或运行时被处理和使用。注解在Java中被广泛用于标记代码，生成文档，代码分析，编译时检查，框架配置等。

基本概念

什么是注解？

注解是添加到Java代码中的元数据，不会直接影响代码的执行。注解可以用于：

  1. 编译时检查：如@Override注解。
  2. 生成文档：如@Deprecated注解。
  3. 运行时处理：如反射机制获取注解信息。

内置注解

Java提供了一些常用的内置注解：

  1. @Override：用于标记方法，表示该方法重写了父类的方法。
  2. @Deprecated：用于标记不建议使用的元素，编译器会发出警告。
  3. @SuppressWarnings：用于抑制编译器警告。

自定义注解

你可以根据需要定义自己的注解，并指定其作用范围和保留策略。

注解的高级应用

元注解

元注解（Meta-Annotations）是用于注解其他注解的注解。Java提供了几种元注解，用于控制自定义注解的行为和使用范围。下面介绍常见的元注解及其作用。

常见元注解

  • @Retention
  • @Target
  • @Documented
  • @Inherited
  • @Repeatable
  1. @Retention

@Retention元注解用于指定注解的保留策略，即注解在什么阶段可用。它有三个可选值：

- RetentionPolicy.SOURCE：注解只在源代码中存在，编译时会被丢弃。
- RetentionPolicy.CLASS：注解在字节码中存在，但在运行时不会被保留。
- RetentionPolicy.RUNTIME：注解在运行时也保留，可以通过反射机制读取。

  1. @Target

@Target元注解用于指定注解可以应用于哪些程序元素。它的参数是一个ElementType数组，可以包括以下值：

  • ElementType.TYPE：应用于类、接口（包括注解类型）或枚举声明。
  • ElementType.FIELD：应用于字段（包括枚举常量）。
  • ElementType.METHOD：应用于方法。
  • ElementType.PARAMETER：应用于方法参数。
  • ElementType.CONSTRUCTOR：应用于构造函数。
  • ElementType.LOCAL_VARIABLE：应用于局部变量。
  • ElementType.ANNOTATION_TYPE：应用于注解类型。
  • ElementType.PACKAGE：应用于包声明。
  • ElementType.TYPE_PARAMETER：应用于类型参数（Java 8引入）。
  • ElementType.TYPE_USE：应用于类型使用（Java 8引入）。

META-INF

META-INF是Java应用程序和库中用于存储元数据的一个特殊目录。它存在于JAR（Java Archive）文件和WAR（Web Application Archive）文件中，用于存放各种配置文件和描述符，帮助Java运行时环境和开发工具了解库或应用的结构和依赖。

META-INF目录的常见内容

  1. MANIFEST.MF：清单文件，包含有关JAR文件的元数据，如版本信息、入口点等。
    services目录：包含服务提供者配置文件，用于SPI（Service Provider Interface）机制。
  2. maven目录：包含Maven相关的元数据，如pom.xml等。
  3. spring.factories：Spring框架使用的配置文件，定义自动配置类等。
