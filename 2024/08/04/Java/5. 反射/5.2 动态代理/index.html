5.2 动态代理

Yuwl 2024-08-04 16:16:13
Categories： Java Tags： java

现在我们来学习 Java 中的动态代理。

Java动态代理的原理主要依赖于反射机制和字节码生成。学习Java的动态代理（Dynamic Proxy）是掌握Java高级编程技术的一个重要方面。动态代理允许你在运行时创建接口的代理实例，这在AOP（面向方面编程）和许多框架（如Spring）中被广泛使用。

基本概念

什么是动态代理？

动态代理是一种设计模式，它允许你在运行时创建一个实现特定接口的代理类，并通过该代理类来控制对实际对象的方法调用。

Java提供了两个核心类来实现动态代理：

  1. java.lang.reflect.Proxy
  2. java.lang.reflect.InvocationHandler

什么时候使用动态代理？

  1. AOP（面向方面编程）：如日志记录、事务管理、性能监控等。
  2. 远程方法调用（RMI）：在分布式系统中代理远程服务。
  3. Mock测试：在单元测试中创建模拟对象。

具体工作机制

  1. 创建InvocationHandler实现类
    InvocationHandler是一个接口，包含一个方法invoke。所有对代理对象的方法调用都会被重定向到InvocationHandler的invoke方法。

  2. 使用Proxy类创建代理对象
    Proxy类的newProxyInstance方法用于创建代理对象。该方法需要三个参数：

  • 类加载器
  • 代理类要实现的接口
  • InvocationHandler实现类的实例
  1. 代理类的生成过程
    当调用Proxy.newProxyInstance时，JVM会做以下工作：
  • 检查接口：检查所有传递的接口是否是有效的接口。
  • 生成代理类：使用字节码生成库（如sun.misc.ProxyGenerator），在运行时生成代理类的字节码。
  • 加载代理类：使用指定的类加载器加载生成的代理类。
  • 实例化代理类：创建代理类的实例，并将InvocationHandler实例与之关联。
  1. 方法调用过程
    当调用代理对象的方法时，方法调用会被转发到InvocationHandler的invoke方法：
  • 代理对象的方法调用：代理对象的方法被调用。
  • invoke方法执行：InvocationHandler的invoke方法被调用。
  • 实际方法调用：在invoke方法中，通过反射调用实际对象的方法。
