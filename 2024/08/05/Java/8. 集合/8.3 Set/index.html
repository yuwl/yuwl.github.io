8.3 Set

Yuwl 2024-08-05 16:07:22
Categories： Java Tags： java

现在我们来学习 Java 中的集合 Set。

Set接口是Java集合框架的一部分，用于存储不重复的元素。常用的Set实现类包括HashSet、LinkedHashSet和TreeSet。我们将重点介绍这三种实现。

HashSet

HashSet基于哈希表实现，存储元素的顺序不保证是插入的顺序。它的主要特点是：

  1. 优点: 查找、插入和删除操作都非常快。
  2. 缺点: 不保证元素的顺序。

HashSet 示例

LinkedHashSet

LinkedHashSet是HashSet的子类，它维护着一个链接列表，记录着元素的插入顺序。它的主要特点是：

  1. 优点: 既具有HashSet的快速操作，又能保证插入顺序。
  2. 缺点: 相比HashSet，内存开销略大。

LinkedHashSet 示例

TreeSet

TreeSet基于红黑树实现，保证元素的自然顺序或者自定义顺序。它的主要特点是：

  1. 优点: 元素是有序的，适用于需要排序操作的场景。
  2. 缺点: 操作相对HashSet和LinkedHashSet稍慢。

TreeSet 示例

TreeSet 自定义排序

通过提供一个自定义的Comparator来定义排序规则。Comparator接口需要实现compare方法。

LinkedHashSet和TreeSet的区别

特性 LinkedHashSet TreeSet
底层数据结构 哈希表 + 双向链表 红黑树（自平衡二叉搜索树）
顺序 插入顺序 自然顺序或自定义顺序
时间复杂度 插入、删除、查找为 O(1) 插入、删除、查找为 O(log n)
是否允许null值 允许 不允许（在比较器中会抛出NullPointerException）
适用场景 需要维持插入顺序且快速访问的场景 需要有序集合且需要排序操作的场景

总结

HashSet: 适合不需要顺序的快速查找、插入和删除操作。
LinkedHashSet: 适合需要维护插入顺序的场景，同时保证快速操作。
TreeSet: 适合需要有序集合的场景，提供自然顺序或自定义顺序。
