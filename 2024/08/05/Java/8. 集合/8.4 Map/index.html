8.4 Map

Yuwl 2024-08-05 16:07:32
Categories： Java Tags： java

现在我们来学习 Java 中的集合 Map。

Map接口是Java集合框架的一部分，用于存储键值对（key-value pairs）。常用的Map实现类包括HashMap、LinkedHashMap和TreeMap。我们将重点介绍这三种实现。

HashMap

HashMap基于哈希表实现，存储键值对的顺序不保证是插入的顺序。它的主要特点是：

  1. 优点: 查找、插入和删除操作都非常快。
  2. 缺点: 不保证键值对的顺序。

5. Hashtable与HashMap的区别

虽然Hashtable和HashMap都实现了Map接口，但它们之间有一些关键的区别：

  • 线程安全：Hashtable是线程安全的，所有方法都被 synchronized修饰，而HashMap不是。
  • null键和null值：Hashtable不允许键或值为null，而HashMap允许一个null键和多个null值。
  • 性能：由于线程安全的开销，Hashtable通常比HashMap慢。
  • 引入时间：Hashtable是Java 1.0中的原始类，HashMap在Java 1.2中引入。

LinkedHashMap

LinkedHashMap是HashMap的子类，它维护着一个链接列表，记录着键值对的插入顺序。它的主要特点是：

  1. 优点: 既具有HashMap的快速操作，又能保证插入顺序。
  2. 缺点: 相比HashMap，内存开销略大。

1. LinkedHashMap 示例

TreeMap

TreeMap基于红黑树实现，保证键值对的自然顺序或者自定义顺序。它的主要特点是：

  1. 优点: 键值对是有序的，适用于需要排序操作的场景。
  2. 缺点: 操作相对HashMap和LinkedHashMap稍慢。

LinkedHashMap 和 TreeMap的区别

特性 LinkedHashMap TreeMap
底层数据结构 哈希表 + 双向链表 红黑树（自平衡二叉搜索树）
顺序 插入顺序或访问顺序 按键的自然顺序或自定义顺序
时间复杂度 插入、删除、查找为 O(1) 插入、删除、查找为 O(log n)
是否允许null键 允许 不允许（在比较器中会抛出NullPointerException）
适用场景 需要维持插入顺序或访问顺序且快速访问的场景 需要有序键值对集合且需要排序操作的场景

总结

  1. HashMap: 适合不需要顺序的快速查找、插入和删除操作。
  2. LinkedHashMap: 适合需要维护插入顺序的场景，同时保证快速操作。
  3. TreeMap: 适合需要有序键值对的场景，提供自然顺序或自定义顺序。
