11.1 JUnit单元测试

Yuwl 2024-08-10 15:21:38
Categories： Java Tags： junit

现在我们来学习 Java 中的 JUnit单元测试。

JUnit 4是一个用于Java编程语言的单元测试框架，允许开发人员编写可重复的测试，以确保代码的正确性和稳定性。JUnit 4引入了许多新特性，使得测试的编写和管理更加方便。

JUnit 4的基本概念

1. 测试用例

测试用例是一个类，包含一个或多个测试方法。每个测试方法都验证代码的特定行为。

2. 注解

JUnit 4使用注解来标识测试方法和生命周期方法，常用的注解包括：

  • @Test：标识一个测试方法。
  • @Before：在每个测试方法之前执行的方法。
  • @After：在每个测试方法之后执行的方法。
  • @BeforeClass：在所有测试方法之前执行一次的静态方法。
  • @AfterClass：在所有测试方法之后执行一次的静态方法。

3. 断言

断言用于检查测试结果是否符合预期，JUnit 4提供了多种断言方法，如assertEquals、assertTrue、assertFalse等。

  • 测试异常：学习如何测试方法是否抛出预期的异常，使用@Test(expected = Exception.class)注解。
  • 参数化测试：了解如何使用JUnit 4的参数化测试功能来运行同一测试多次，使用不同的输入。

JUnit 4断言

JUnit 4提供了多种断言方法，用于验证测试结果是否符合预期。断言是单元测试的核心部分，能够帮助开发人员判断代码的正确性。以下是一些常用的JUnit 4断言方法及其用途：

常用断言方法

方法 描述
assertNull(Object object) 检查对象是否为null。
assertNotNull(Object object) 检查对象是否不为null。
assertEquals(long expected, long actual) 检查两个long类型的值是否相等。
assertEquals(double expected, double actual, double delta) 检查两个double值是否相等，允许一定的误差范围。
assertFalse(boolean condition) 检查条件是否为false。
assertTrue(boolean condition) 检查条件是否为true。
assertSame(Object expected, Object actual) 检查两个对象引用是否指向同一个对象。
assertNotSame(Object unexpected, Object actual) 检查两个对象引用是否不指向同一个对象。

在JUnit 4中，你可以通过以下两种主要方式来测试异常：

  • 使用@Test(expected = Exception.class)注解：这种方式用于指定测试方法预期抛出的异常类型。
  • 使用try-catch块：这种方式允许你在测试中捕获异常并进行更详细的断言。

JUnit 4参数化测试

JUnit 4参数化测试的步骤

1. 添加JUnit 4依赖

如果你还没有在项目中添加JUnit 4的依赖，请确保在pom.xml（对于Maven）或build.gradle（对于Gradle）中添加JUnit 4的依赖。

2. 创建测试类

在测试类中，使用@RunWith(Parameterized.class)注解来标识该类为参数化测试类。

3. 创建参数化数据

使用@Parameterized.Parameters注解创建一个公共静态方法，该方法返回一个Collection对象，包含多个测试数据集。

4. 创建构造函数

创建一个构造函数，接受每组测试数据的参数，并将其分配给实例变量。

5. 编写测试方法

编写测试方法，使用实例变量作为输入参数进行测试。
