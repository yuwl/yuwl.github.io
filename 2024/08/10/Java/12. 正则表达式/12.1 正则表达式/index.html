12.1 正则表达式

Yuwl 2024-08-10 15:46:50
Categories： Java Tags： regex

现在我们来学习 Java 中的 正则表达式。

正则表达式（Regular Expressions，简称regex）是一种用于匹配字符串模式的强大工具。

正则表达式

1. 基本概念

正则表达式是一种模式匹配的语法，用于查找、替换、提取和验证字符串。正则表达式由普通字符（如字母和数字）和特殊字符（称为元字符）组成。

2. 设置开发环境

确保你的Java开发环境已经配置好，可以使用正则表达式。Java中的正则表达式主要由java.util.regex包中的类支持，如Pattern和Matcher。

3. 基本语法和示例

匹配字符

  • .: 匹配任何单个字符（除换行符外）。
  • \d: 匹配任何数字，相当于[0-9]。
  • \D: 匹配任何非数字字符，相当于[^0-9]。
  • \w: 匹配任何单词字符（字母、数字、下划线），相当于[a-zA-Z0-9_]。
  • \W: 匹配任何非单词字符，相当于[^a-zA-Z0-9_]。
  • \s: 匹配任何空白字符（空格、制表符、换页符等）。
  • \S: 匹配任何非空白字符。

重复匹配

  • *: 匹配前面的子表达式零次或多次。
  • +: 匹配前面的子表达式一次或多次。
  • ?: 匹配前面的子表达式零次或一次。
  • {n}: 匹配前面的子表达式恰好n次。
  • {n,}: 匹配前面的子表达式至少n次。
  • {n,m}: 匹配前面的子表达式至少n次但不超过m次。

位置匹配

  • ^: 匹配字符串的开头。
  • $: 匹配字符串的结尾。
  • \b: 匹配一个单词边界。
  • \B: 匹配非单词边界。

分组和选择

  • (...): 捕获组。
  • (?:...): 非捕获组。
  • |: 或操作符，匹配它前面或后面的子表达式。

匹配规则

1. 基本字符匹配

  • 普通字符：直接匹配字符自身。例如，正则表达式abc会匹配字符串abc。
  • 点号（.）：匹配除换行符之外的任意单个字符。例如，正则表达式a.c会匹配abc、a1c、a-c等。

2. 字符类

  • 方括号（[]）：匹配方括号中的任意一个字符。例如，正则表达式[abc]会匹配a、b或c中的任意一个。
  • 字符范围：方括号中的字符可以使用范围。例如，正则表达式[a-z]会匹配所有小写字母，[0-9]会匹配所有数字。
  • 排除字符类（[^…]）：匹配不在方括号中的任意字符。例如，正则表达式[^0-9]会匹配除数字之外的任意字符。

3. 预定义字符类

  • \d：匹配任何数字，相当于[0-9]。
  • \D：匹配任何非数字字符，相当于[^0-9]。
  • \w：匹配任何单词字符（字母、数字、下划线），相当于[a-zA-Z0-9_]。
  • \W：匹配任何非单词字符，相当于[^a-zA-Z0-9_]。
  • \s：匹配任何空白字符（空格、制表符、换页符等）。
  • \S：匹配任何非空白字符。

4. 重复匹配

  • *：匹配前面的子表达式零次或多次。例如，正则表达式ab*会匹配a、ab、abb等。
  • +：匹配前面的子表达式一次或多次。例如，正则表达式ab+会匹配ab、abb、abbb等，但不匹配a。
  • ?：匹配前面的子表达式零次或一次。例如，正则表达式ab?会匹配a或ab。
  • {n}：匹配前面的子表达式恰好n次。例如，正则表达式a{3}会匹配aaa。
  • {n,}：匹配前面的子表达式至少n次。例如，正则表达式a{2,}会匹配aa、aaa、aaaa等。
  • {n,m}：匹配前面的子表达式至少n次但不超过m次。例如，正则表达式a{2,4}会匹配aa、aaa或aaaa。

5. 位置匹配

  • ^：匹配字符串的开头。例如，正则表达式^abc会匹配以abc开头的字符串。
  • $：匹配字符串的结尾。例如，正则表达式abc$会匹配以abc结尾的字符串。
  • \b：匹配一个单词边界。例如，正则表达式\bword\b会匹配独立的单词word。
  • \B：匹配非单词边界。例如，正则表达式\Bword\B不会匹配独立的单词word，而会匹配如password中的word部分。

6. 分组和选择

  • 圆括号（(…)）：用于分组。分组允许将多个字符视为一个单元。例如，正则表达式(abc)+会匹配abc、abcabc等。
  • 管道符号（|）：表示选择，匹配其左右任意一个子表达式。例如，正则表达式a|b会匹配a或b。
  • 非捕获组（(?:…)）：用于分组但不捕获。例如，正则表达式(?:abc)+会匹配abc、abcabc等，但不会捕获这些匹配。

复杂匹配规则

1. 零宽断言

零宽断言用于匹配某个位置，但不包含在匹配结果中。这在一些特定的匹配需求中非常有用。

  • 正向先行断言（(?=…)）：要求后面的子表达式必须匹配，但不包含在匹配结果中。例如，正则表达式\d(?=\D)会匹配一个数字，且该数字后面必须跟着一个非数字字符。
  • 负向先行断言（(?!…)）：要求后面的子表达式不能匹配。例如，正则表达式\d(?!\d)会匹配一个数字，且该数字后面不能跟着另一个数字。
  • 正向后发断言（(?<=…)）：要求前面的子表达式必须匹配，但不包含在匹配结果中。例如，正则表达式(?<=\d)\D会匹配一个非数字字符，且该字符前面必须是一个数字。
  • 负向后发断言（(?<!…)）：要求前面的子表达式不能匹配。例如，正则表达式(?<!\d)\D会匹配一个非数字字符，且该字符前面不能是一个数字。

2. 捕获组和反向引用

捕获组用于将部分匹配的文本保存起来，以便在正则表达式或替换字符串中再次使用。

  • 捕获组（(…)）：将匹配的子表达式保存到一个临时区域。例如，正则表达式(\d{3})-(\d{2})-(\d{4})会捕获三个组，分别是区号、前缀和后缀。
  • 反向引用（\n）：在同一个正则表达式中引用前面的捕获组，其中n是捕获组的编号。例如，正则表达式(\d{2})-(\d{2})-\1会匹配类似12-34-12的字符串，其中最后一个数字组与第一个捕获组相同。

3. 非捕获组

非捕获组用于分组但不捕获匹配的文本。它们在需要分组但不希望存储匹配结果时非常有用。

  • 非捕获组（(?:…)）：例如，正则表达式(?:abc)+会匹配一个或多个abc，但不会捕获匹配的文本。

非贪婪匹配

非贪婪匹配（也称为懒惰匹配）是正则表达式中的一种匹配策略，它会匹配尽可能少的字符，以便满足整个表达式的匹配。与之相对的是贪婪匹配，它会匹配尽可能多的字符。非贪婪匹配在处理嵌套结构或需要精确控制匹配长度时非常有用。

非贪婪匹配的语法

在正则表达式中，非贪婪匹配的量词是在贪婪量词后面加上一个问号?，例如：

  • *?：匹配前面的子表达式零次或多次，但尽可能少地匹配。
  • +?：匹配前面的子表达式一次或多次，但尽可能少地匹配。
  • ??：匹配前面的子表达式零次或一次，但尽可能少地匹配。
  • {n,m}?：匹配前面的子表达式至少n次，但不超过m次，尽可能少地匹配。
