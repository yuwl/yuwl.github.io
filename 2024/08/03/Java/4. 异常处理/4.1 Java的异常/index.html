4.1 Java的异常

Yuwl 2024-08-03 16:43:16
Categories： Java Tags： java

现在我们来学习 Java 中的异常。

什么是异常？

异常是在程序执行过程中发生的一个事件，它会中断程序的正常流程。例如，当你试图除以零或访问一个不存在的文件时，就会发生异常。

Java 异常层次结构

Java 中的所有异常都是 Throwable 类的子类。Throwable 有两个主要的子类：Error 和 Exception。

  1. Error：表示严重的问题，通常是不可恢复的。例如 OutOfMemoryError。
  2. Exception：表示可以被处理的问题。又分为两类：
    • 检查型异常（Checked Exceptions）：必须被显式处理或声明抛出。
    • 非检查型异常（Unchecked Exceptions）：运行时异常，不强制处理。

Error和Exception的区别:

  1. 继承关系:
  • Error是Throwable的直接子类
  • Exception也是Throwable的直接子类
  1. 性质:
  • Error表示严重的问题,通常是不可恢复的系统级错误
  • Exception表示程序可能出现的异常情况,通常是可以被捕获和处理的
  1. 处理方式:
  • Error通常不应该被捕获,因为它代表了无法恢复的错误状态
  • Exception应该被捕获和适当处理,以保证程序的健壮性
  1. 常见例子:
  • Error: OutOfMemoryError, StackOverflowError等
  • Exception: IOException, SQLException, NullPointerException等
  1. 检查性:
  • Error属于非检查型(Unchecked)
  • Exception分为检查型(Checked)和非检查型(Unchecked),RuntimeException及其子类属于非检查型
  1. 发生时机:
  • Error通常在运行时环境出现问题时发生
  • Exception可能在编译时检测(Checked Exception)或运行时发生(Unchecked Exception)
  1. 恢复可能性:
  • Error通常是不可恢复的
  • Exception通常可以通过适当的异常处理来恢复程序的正常执行

检查型异常（Checked Exception）和非检查型异常（Unchecked Exception）

检查型异常（Checked Exception）

  1. 定义
    检查型异常是指在编译时由编译器强制检查的异常。如果代码中可能抛出检查型异常，必须显式地捕获或声明抛出这些异常，否则编译不会通过。
  2. 特点
  • 编译时检查：必须在编译时处理（使用 try-catch 块捕获或在方法签名中使用 throws 声明）。
  • 强制处理：调用者必须处理这些异常，否则编译器会报错。
  • 常见检查型异常：如 IOException、SQLException、ClassNotFoundException 等。
  1. 示例

非检查型异常（Unchecked Exception）

  1. 定义
    非检查型异常是指在编译时不强制检查的异常。这些异常通常是编程错误，如逻辑错误或不正确的使用API。
  2. 特点
  • 运行时检查：在编译时不强制处理，可以选择捕获或不处理。
  • 非强制处理：编译器不会强制要求处理这些异常。
  • 常见非检查型异常：如 NullPointerException、- ArrayIndexOutOfBoundsException、ArithmeticException 等。

自定义异常。

1. 什么是自定义异常？

自定义异常是指程序员自己定义的异常类，通常用于表示特定的业务逻辑错误或应用程序特有的错误情况。

2. 为什么需要自定义异常？

  1. 提供更具体的错误信息
  2. 使代码更易读和维护
  3. 更好地表达业务逻辑
  4. 区分不同类型的错误

3. 如何创建自定义异常

创建自定义异常通常包括以下步骤：

  1. 创建一个新类，继承自 Exception（检查型异常）或 RuntimeException（非检查型异常）
  2. 提供构造函数
  3. 根据需要重写方法（如 getMessage()）

5. 检查型异常 vs 非检查型异常

  1. 如果你的自定义异常继承自 Exception，它将是一个检查型异常。
  2. 如果继承自 RuntimeException，它将是一个非检查型异常。
  3. 选择取决于你希望调用者是否必须显式处理这个异常。
