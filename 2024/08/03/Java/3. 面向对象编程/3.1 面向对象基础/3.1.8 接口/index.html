3.1.8 接口

Yuwl 2024-08-03 10:30:50
Categories： Java Tags： java

现在我们来学习 Java 面向对象中的接口。

什么是接口？

接口是一种引用类型，它是抽象方法的集合。接口可以包含常量、方法签名、默认方法、静态方法和嵌套类型。接口不能包含实例字段，也不能包含构造方法。

接口的特点

  1. 使用 interface 关键字声明。
  2. 接口中的方法默认是 public 和 abstract 的（在Java 8之前）。
  3. 接口中的变量默认是 public static final 的。
  4. 一个类可以实现多个接口。
  5. 接口可以继承其他接口。

接口的优点

  1. 解耦：接口提供了一种实现代码解耦的方式，使代码更加灵活和可扩展。
  2. 多态性：通过接口引用，可以实现多态。
  3. 代码重用：通过实现多个接口，可以实现代码重用。
  4. 设计灵活性：接口使得设计更加灵活，可以轻松地替换实现。

接口 vs 抽象类

  1. 抽象类可以有构造方法，接口不能。
  2. 抽象类可以包含非抽象方法，接口中的方法默认是抽象的（Java 8之前）。
  3. 抽象类可以有成员变量，接口中的变量默认是 public static final 的。
  4. 一个类只能继承一个抽象类，但可以实现多个接口。

使用接口的场景

  1. 当你需要定义一组不相关的类共享的方法，但不需要共享实现。
  2. 当你需要实现多重继承。
  3. 当你需要定义行为的契约，而不关心具体的实现。
