3.1.7 抽象类

Yuwl 2024-08-03 10:30:42
Categories： Java Tags： java

现在我们来学习 Java 面向对象中的抽象类。

什么是抽象类？

抽象类是一种不能被实例化的类，它主要用于定义子类的通用特征。抽象类可以包含抽象方法（没有实现的方法）和具体方法（有实现的方法）。

抽象类的特点

  1. 使用 abstract 关键字声明。
  2. 可以包含抽象方法和非抽象方法。
  3. 不能被实例化。
  4. 可以有构造方法。
  5. 可以包含成员变量。
  6. 可以包含静态方法。

抽象类的优点

  1. 代码重用：抽象类可以包含通用的实现，子类可以直接使用。
  2. 强制实现：子类必须实现抽象类中的所有抽象方法。
  3. 设计灵活性：可以定义必须由子类实现的方法，同时提供一些通用的实现。
  4. 多态性：可以通过抽象类类型的引用来引用其子类对象。

抽象类 vs 接口

  1. 抽象类可以有构造方法，接口不能。
  2. 抽象类可以包含非抽象方法，接口中的方法默认是抽象的（Java 8之前）。
  3. 抽象类可以有成员变量，接口中的变量默认是 public static final 的。
  4. 一个类只能继承一个抽象类，但可以实现多个接口。

使用抽象类的场景

  1. 当你想要在几个相关的类之间共享代码。
  2. 当你期望扩展抽象类的类有许多共同的方法或字段，或者需要非public的访问修饰符。
  3. 当你想声明非静态或非常量字段。
