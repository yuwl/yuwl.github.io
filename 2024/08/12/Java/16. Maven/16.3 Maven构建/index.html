16.3 Maven构建

Yuwl 2024-08-12 22:11:12
Categories： Java Tags： Maven

现在我们来学习 Maven构建。

构建和打包

使用 Maven 进行项目的构建、编译、测试和打包是 Maven 的核心功能之一。Maven 通过定义标准化的生命周期、阶段和插件，使得构建过程变得简单且一致。下面将详细介绍如何使用 Maven 进行这些操作。

1. 项目构建

项目构建指的是从项目源代码生成可运行的应用程序或库的过程，包括编译代码、运行测试、打包和部署等步骤。

命令:

mvn install: 这是最常用的命令，它会触发 default 生命周期的所有阶段，从 validate 到 install，最后将生成的包安装到本地仓库。

执行效果:

  • 验证项目结构和 POM 文件 (validate)。
  • 编译源代码 (compile)。
  • 运行单元测试 (test)。
  • 打包编译后的代码 (package)。
  • 在本地仓库中安装包 (install)。

2. 编译源代码

Maven 编译源代码的过程通过 compile 阶段实现，它将 src/main/java 目录下的所有 Java 源文件编译到 target/classes 目录。

命令:

mvn compile: 仅编译源代码，而不进行打包或其他操作。

执行效果:

将 src/main/java 中的代码编译到 target/classes。

3. 运行测试

Maven 提供了对单元测试的支持，通过 test 阶段，Maven 使用 JUnit 或 TestNG 等框架来执行测试代码，通常位于 src/test/java 目录。

命令:

mvn test: 编译项目并运行测试代码。

执行效果:

  • 编译测试代码 (compile 阶段)。
  • 运行所有测试，并生成测试报告。

4. 打包项目

打包是 Maven 项目构建的核心步骤，它会根据项目类型生成相应的包，例如 JAR、WAR、或 EAR 文件。

命令:

mvn package: 编译代码并将其打包为 JAR、WAR 或其他指定格式的文件。

执行效果:

将编译后的代码和资源文件打包到 target 目录中的 JAR 或 WAR 文件中。

JAR 包:

如果是一个标准的 Java 项目，Maven 默认会生成一个 JAR 包，包含编译后的类文件、依赖库和资源文件。

5. 安装到本地仓库

安装是将打包生成的 JAR 或 WAR 文件放置到本地 Maven 仓库中的过程，通常用于本地开发或其他项目的依赖。

命令:

mvn install: 将打包好的文件安装到本地仓库 (~/.m2/repository)，供其他 Maven 项目使用。

执行效果:

将打包后的文件安装到本地仓库中，供本地开发和依赖解析使用。

6. 部署到远程仓库

部署是将打包后的文件发布到远程仓库的过程，通常用于发布到公司内部的私有仓库或公共仓库。

命令:

mvn deploy: 将打包后的文件部署到远程仓库。

7. 跳过测试

有时在构建过程中，你可能希望跳过测试，以加快构建速度。你可以使用 -DskipTests 选项。

命令:

mvn package -DskipTests: 跳过测试，直接进行打包。

Maven 插件

Maven 插件是 Maven 构建过程的核心组件，它们负责执行特定的任务，例如编译代码、运行测试、打包项目等。每个插件包含多个目标（goals），这些目标可以在 Maven 的不同生命周期阶段中绑定执行。以下是常见的 Maven 插件及其用法，包括编译插件、测试插件和打包插件。

基本配置

  • source: 指定源代码的 Java 版本（例如 1.8）。
  • target: 指定编译生成的字节码版本（例如 1.8）。

  • pom.xml: 项目的 Maven 配置文件。
