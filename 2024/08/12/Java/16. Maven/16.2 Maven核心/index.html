依赖管理

在 Maven 中，依赖管理是一个关键部分，它帮助你定义和管理项目所需的外部库。通过 POM 文件中的 dependencies 元素，你可以轻松添加、排除和管理依赖。下面我们来详细介绍这些操作。

1. 添加依赖

添加依赖是通过在 POM 文件的 dependencies 元素中定义依赖项来完成的。每个依赖项通常包括 groupId, artifactId, 和 version。

2. 依赖范围 (scope)

scope 定义了依赖的使用范围，Maven 提供了几种不同的范围：

  • compile: 默认范围，依赖在编译、测试和运行时均可用。

  • provided: 依赖在编译和测试时可用，但在运行时由外部环境提供。

  • runtime: 依赖在运行时和测试时可用，但在编译时不可用。

  • test: 依赖仅在测试编译和测试运行时可用。

  • system: 依赖是由本地系统提供的，并且必须使用 systemPath 指定路径。

  • import: 用于引入 BOM（Bill of Materials）来管理依赖版本。

Maven 仓库

Maven 的仓库系统是依赖管理的核心部分，它决定了 Maven 如何下载、存储和查找依赖。Maven 仓库主要分为三种类型：中央仓库、本地仓库和私有仓库。了解它们的作用和配置对于高效使用 Maven 至关重要。

1. 中央仓库（Central Repository）

中央仓库是 Maven 官方维护的公共仓库，包含了大量的开源项目和库。每次构建项目时，Maven 会首先在中央仓库中查找依赖，如果依赖没有在本地找到，就会从中央仓库下载。

默认的中央仓库 URL:

如何使用:

中央仓库默认启用，通常不需要额外配置。当你在 pom.xml 文件中声明依赖时，Maven 会自动尝试从中央仓库下载这些依赖。

2. 本地仓库（Local Repository）

本地仓库是 Maven 在你本地机器上创建的一个目录，用于缓存和存储从中央仓库或私有仓库下载的依赖。每次构建时，Maven 会首先在本地仓库查找依赖，如果找到就直接使用，而不需要重新下载。

本地仓库的默认路径:

  • Windows: C:\Users<your_username>.m2\repository
  • Linux/Mac: /Users//.m2/repository 或 /home//.m2/repository

配置本地仓库路径:

如果你想更改本地仓库的位置，可以通过 Maven 的 settings.xml 文件进行配置。

settings.xml 文件路径:

  • Windows: C:\Users<your_username>.m2\settings.xml
  • Linux/Mac: /Users//.m2/settings.xml 或 /home//.m2/settings.xml

3. 私有仓库（Private Repository）

私有仓库通常由公司或组织内部搭建，用于存储私有的、公司内部的库或组件。这些库可能是公司开发的专有代码或定制的第三方依赖，不能直接发布到中央仓库。

私有仓库的用途:

  • 存储公司内部开发的库。
  • 缓存第三方依赖，以减少外部网络依赖。
  • 提供对私有组件的版本控制和管理。

常见的私有仓库解决方案:

  • Nexus Repository Manager
  • Artifactory
  • Apache Archiva

4. 仓库的优先级和搜索顺序

Maven 在解析依赖时，遵循以下优先级和搜索顺序：

  • 本地仓库: 首先查找本地仓库，检查依赖是否已经存在。
  • 私有仓库: 如果依赖不在本地仓库中，Maven 会按照配置的顺序依次查询私有仓库。
  • 中央仓库: 最后，Maven 会查询中央仓库，获取所需的依赖。
