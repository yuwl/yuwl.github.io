14.7 Fork-Join

Yuwl 2024-08-11 16:14:16
Categories： Java Tags： 多线程

现在我们来学习 Java 中一个功能强大的框架 Fork/Join。

Fork/Join 框架 是 Java 7 引入的一个框架，用于并行处理任务。它是 java.util.concurrent 包的一部分，旨在充分利用多核处理器的能力。Fork/Join 框架将任务分解为更小的子任务，然后将这些子任务分配到多个处理器内核上进行并行处理。

1. 基本概念

  • Fork：将一个大任务拆分成多个子任务，每个子任务再进一步拆分，直到任务足够小以便直接解决。
  • Join：将多个子任务的结果合并成最终的结果。

Fork/Join 框架基于 工作窃取算法，即当一个线程完成了它自己的任务时，它会从其他繁忙的线程中窃取任务来执行，从而提高 CPU 的利用率。

2. Fork/Join 框架的核心类

  • ForkJoinTask：一个抽象类，表示可以并行执行的任务。常见的子类包括 RecursiveTask（有返回值的任务）和 RecursiveAction（无返回值的任务）。

  • ForkJoinPool：Fork/Join 框架的执行器，用于管理和协调任务的执行。它是一个专门用于处理 ForkJoinTask 的线程池。

3. 使用步骤

3.1 创建任务类

继承 RecursiveTask 或 RecursiveAction 类，实现 compute 方法。compute 方法负责将任务分解（fork）并合并结果（join）。

3.2 提交任务

将任务提交到 ForkJoinPool 中执行。

3.3 获取结果

使用 join() 方法获取任务的执行结果。

4. 示例：使用 Fork/Join 框架计算数组的元素和

假设你需要计算一个大数组的元素之和，使用 Fork/Join 框架可以将数组分成若干小块，并行计算每个小块的和，最后再将这些小块的结果汇总。

5. 分析代码

  • 任务拆分：在 compute 方法中，如果任务的大小（数组长度）小于阈值（THRESHOLD），则直接计算。否则，将任务分成两部分并行执行。

  • fork() 方法：用于异步执行子任务。这里 leftTask.fork() 意味着把左边的子任务放到工作队列中，并立即返回。

  • compute() 方法：同步执行右边的子任务。在本例中，rightTask.compute() 是直接计算右边的任务。

  • join() 方法：用于等待任务的完成并返回结果。在这里，leftResult = leftTask.join() 是等待左边的任务完成，并返回其结果。

6. ForkJoinPool 的使用

ForkJoinPool 是 Fork/Join 框架的执行器，它管理和调度 Fork/Join 任务。你可以通过以下几种方式使用 ForkJoinPool：

  • invoke()：同步调用，提交任务并等待完成返回结果。
  • submit()：异步调用，提交任务并返回 ForkJoinTask 对象，后者可用于进一步操作（如 join()）。
  • execute()：异步调用，提交任务但不返回结果。

7. Fork/Join 框架的优点

  • 并行化：能够充分利用多核 CPU 的计算能力，显著提高性能。
  • 灵活性：通过任务拆分与合并，可以处理各种复杂的计算任务。
  • 自动负载均衡：工作窃取算法能够有效地分配任务，避免某些线程过于繁忙而另一些线程空闲。

8. 注意事项

  • 任务拆分的粒度：合理选择任务拆分的阈值，过小的粒度会增加任务管理的开销，过大的粒度则无法充分利用并行处理的优势。

  • 递归深度：递归深度过深可能会导致栈溢出，需要谨慎处理大规模任务。

  • ForkJoinPool 的大小：默认为可用处理器的数量，你可以根据需求调整线程池的大小。
