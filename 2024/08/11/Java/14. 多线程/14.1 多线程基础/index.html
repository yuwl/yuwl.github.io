14.1 多线程基础

Yuwl 2024-08-11 09:10:47
Categories： Java Tags： 多线程

现在我们来学习 Java 中的 多线程。

多线程是Java编程中一个非常重要的概念，能够让程序同时执行多个任务，提高效率和响应速度。

基本概念

线程的基本概念

  • 线程：线程是程序执行的最小单位。一个Java程序可以包含多个线程，它们可以并发执行。
  • 并发：指多个线程在同一时间段内执行，可能是交替执行，也可能是同时执行。
  • 多线程的优势：
    • 提高程序的效率
    • 改善用户体验（例如，UI线程可以保持响应，而后台线程处理耗时操作）

与进程的区别

进程与线程的定义

  • 进程：进程是一个正在执行的程序的实例，是操作系统资源分配的基本单位。每个进程都有独立的内存空间和系统资源，进程之间的通信相对复杂，通常需要通过进程间通信（IPC）机制如管道、消息队列等进行。
  • 线程：线程是进程中的一个执行单元，是CPU调度的基本单位。一个进程可以包含多个线程，这些线程共享进程的内存空间和资源，但每个线程有自己的栈和程序计数器。线程之间的通信相对简单，因为它们可以直接访问共享的内存空间。

主要区别

  1. 资源分配：
  • 进程拥有独立的地址空间，系统为每个进程分配资源。
  • 线程共享同一进程的地址空间和资源。
  1. 开销：
  • 进程切换的开销较大，因为需要保存和加载进程的上下文。
  • 线程切换的开销较小，因为线程共享相同的地址空间，切换时只需保存和加载线程的上下文。
  1. 稳定性：
  • 进程之间相互独立，一个进程崩溃不会影响其他进程。
  • 如果一个线程崩溃，整个进程也会受到影响。
  1. 并发性：
  • 进程之间的并发性较低，通常需要通过IPC进行通信。
  • 线程之间的并发性较高，可以直接共享数据和资源。
  1. 创建和销毁：
  • 创建和销毁进程的代价较高。
  • 创建和销毁线程的代价较低。

1. 同步方法

描述: 在方法上使用synchronized关键字，这个方法在同一时间只能被一个线程访问。

描述: 使用synchronized修饰静态方法或以类对象作为锁，可以在类级别上实现同步。

volatile

volatile 关键字是 Java 中的一种轻量级同步机制，用于确保多个线程访问共享变量时的可见性。它可以用于变量声明，告诉编译器和运行时，不要对这个变量进行缓存，每次直接从内存中读取最新的值。

1. volatile 的作用

volatile 关键字有两个主要作用：

  • 保证变量的可见性：当一个线程修改了 volatile 变量的值，其他线程能够立即看到这个修改。这是通过禁止 CPU 缓存和线程本地缓存来实现的。
  • 防止指令重排序：使用 volatile 还会禁止 JVM 对这个变量的读写操作进行指令重排序，确保操作的顺序性。

2. volatile 的使用场景

volatile 适用于以下情况：

  • 变量被多个线程共享，且各线程读取和写入变量值时不依赖变量的当前值（即不需要进行复杂的计算）。
  • 变量在单一线程中被写入，但在多个线程中被读取。
  • 对变量的操作是独立的，即不会受到其他变量或操作的影响。

3. volatile 的局限性

  • volatile 不能替代 synchronized。它仅适用于保证变量的可见性，不能保证复合操作（如 i++）的原子性。如果多个线程需要对一个变量进行复杂操作，仍然需要使用 synchronized 或 Lock。
  • volatile 只适用于轻量级同步，不适合需要锁的场景。

死锁

死锁 是指在多线程环境中，两个或多个线程因为相互等待对方释放资源，而导致无法继续执行的现象。简单来说，死锁是一种线程永远卡住的状态。

1. 死锁的基本概念

在 Java 中，死锁通常发生在以下场景：

  • 线程 A 拿到锁 1，等待锁 2；同时线程 B 拿到锁 2，等待锁 1。
  • 因为线程 A 和 B 都在等待对方释放锁，这就导致了死锁。

2. 死锁的四个必要条件

死锁的发生需要满足以下四个条件：

  • 互斥条件：资源不能被共享，只能被一个线程占用。
  • 持有并等待条件：线程持有一个资源的同时，等待其他资源。
  • 不可剥夺条件：线程持有的资源不能被强制剥夺，只能由线程主动释放。
  • 循环等待条件：两个或多个线程形成资源循环等待链。

4. 如何避免死锁

虽然死锁可能是个复杂的问题，但通过一些设计和编程技巧，可以减少死锁发生的可能性。

4.1 避免嵌套锁

尽量避免嵌套锁的情况，如果需要多把锁，尽可能在同一个方法中获取所有锁。

4.2 使用锁顺序

确保多个线程获取锁的顺序一致。例如，如果线程 A 先获取 lock1，再获取 lock2，那么线程 B 也应按相同的顺序获取锁。

4.3 使用 tryLock 机制

使用 Lock 和 ReentrantLock 的 tryLock 方法来尝试获取锁，如果锁不可用，则可以避免等待并采取其他措施。
