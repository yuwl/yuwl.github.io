15.1 网络编程基础

Yuwl 2024-08-11 20:15:38
Categories： Java Tags： 网络编程

现在我们来学习 Java 中的 网络编程。

网络编程 是 Java 的一个重要领域，它涉及到如何在计算机之间通过网络发送和接收数据。Java 提供了丰富的 API 来支持网络编程，特别是基于 TCP/IP 协议的编程。

基本概念

  • IP 地址：用于标识网络中的计算机。例如，192.168.1.1 是一个典型的 IP 地址。

  • 端口号：用于标识计算机上的特定进程。每个进程绑定到一个特定的端口号，多个进程可以共享同一个 IP 地址，但每个进程都有不同的端口号。

  • 套接字 (Socket)：套接字是网络编程的核心，它是通信的端点，负责在网络上发送和接收数据。

  • TCP 和 UDP：两种常见的传输层协议。TCP 是面向连接的，提供可靠的数据传输；UDP 是无连接的，适用于快速传输但不保证可靠性的场景。

TCP 编程

使用 TCP 进行网络通信是网络编程中常见的任务，适用于需要可靠数据传输的场景。下面是如何在 Java 中实现基于 TCP 的简单客户端-服务器通信。

1. 基本概念

  • TCP（传输控制协议）：TCP 是一种面向连接的协议，保证数据的可靠传输。它在通信前要先建立连接，数据传输完成后再关闭连接。

  • 服务器：在特定端口上监听客户端连接请求，并与客户端进行数据通信。

  • 客户端：连接服务器，发送请求并接收服务器的响应。

2. 实现步骤

2.1 服务器端实现

服务器端使用 ServerSocket 类来监听特定端口，并使用 Socket 类与客户端通信。

步骤：

  • 创建 ServerSocket，指定监听的端口号。
  • 调用 accept() 方法等待客户端连接。
  • 获取 Socket 对象的输入输出流，用于接收和发送数据。
  • 处理完数据后，关闭连接。 2.2 客户端实现

客户端使用 Socket 类连接到服务器，并通过输入输出流与服务器进行数据通信。

步骤：

  • 创建 Socket 对象，指定服务器的 IP 地址和端口号。
  • 获取 Socket 对象的输入输出流，用于发送和接收数据。
  • 处理完数据后，关闭连接。
  1. 运行示例

  • 启动服务器：首先运行 TCPServer 类。这会启动服务器并开始在指定端口上监听客户端连接。

  • 启动客户端：然后运行 TCPClient 类。客户端会连接到服务器，并可以在控制台输入消息发送到服务器。

  • 交互：服务器接收到客户端的消息后，会回显该消息。客户端会显示服务器的响应。

UDP 编程

UDP（用户数据报协议）是一种无连接的网络协议，适用于需要快速传输但不要求可靠性的场景，如实时视频、音频传输或在线游戏。与 TCP 不同，UDP 不会保证数据包的顺序和可靠性，但它具有更低的延迟和开销。

1. 基本概念

  • UDP（用户数据报协议）：一种无连接的传输协议，不保证数据包的顺序和完整性，但具有快速传输的优点。

  • 数据报（Datagram）：UDP 协议下的数据包，包含发送方、接收方、数据内容等信息。

  • DatagramSocket：用于发送和接收数据报的套接字类。

  • DatagramPacket：封装了发送和接收的数据报信息。

2. 实现步骤

2.1 服务器端实现

服务器端使用 DatagramSocket 类接收来自客户端的数据报。

步骤：

  • 创建 DatagramSocket，绑定到指定端口。
  • 创建 DatagramPacket，用于接收数据报。
  • 调用 receive() 方法接收数据报。
  • 处理接收到的数据，并根据需要发送响应。

2.2 客户端实现

客户端使用 DatagramSocket 类发送数据报到服务器，并接收服务器的响应。

步骤：

  • 创建 DatagramSocket。
  • 创建 DatagramPacket，包含要发送的数据、服务器的地址和端口。
  • 调用 send() 方法发送数据报。
  • 根据需要接收服务器的响应数据报。
  1. 运行示例

  • 启动服务器：首先运行 UDPServer 类。这会启动服务器并在指定端口上监听来自客户端的数据报。

  • 启动客户端：然后运行 UDPClient 类。客户端将向服务器发送消息，并接收服务器的响应。

  • 交互：客户端输入的消息将通过 UDP 发送到服务器，服务器接收到消息后会回显消息内容。

高级-多线程服务器

在高级网络编程中，构建一个多线程服务器是一个常见的任务。多线程服务器能够同时处理多个客户端的请求，适用于高并发场景，如网络游戏、聊天服务器、文件服务器等。

1. 基本概念

  • 多线程服务器：服务器在接收到客户端连接请求时，为每个连接创建一个新的线程来处理通信。这样，服务器可以同时处理多个客户端的请求，而不会因为一个客户端的请求阻塞其他客户端的访问。

  • 线程池：为了避免频繁创建和销毁线程带来的开销，服务器通常使用线程池来管理线程。线程池能够重用已有的线程，提升服务器的性能和响应速度。

2. 实现步骤

2.1 服务器端实现

服务器端在接收到客户端连接后，会为每个连接创建一个新的线程来处理请求。为了提高性能，可以使用 ThreadPoolExecutor 来管理线程池。

步骤：

  • 创建 ServerSocket，指定服务器监听的端口。
  • 使用 ExecutorService 创建线程池，用于管理处理客户端请求的线程。
  • 进入主循环，等待客户端连接。
  • 一旦接受到客户端连接，提交任务到线程池，由线程池中的线程处理客户端请求。
  • 线程处理完请求后，关闭连接。

高级-非阻塞 IO

非阻塞 I/O（Non-blocking I/O）是一种高级网络编程技术，允许在不阻塞应用程序的情况下处理 I/O 操作。相比传统的阻塞 I/O 模型，非阻塞 I/O 提供了更高的并发性能和资源利用率，尤其在高负载和多连接的场景下非常有用。

1. 基本概念

  • 阻塞 I/O：在进行 I/O 操作时，如果数据没有准备好，操作将一直等待，直到数据可用。这种模式简单易用，但在高并发情况下，性能可能会下降。

  • 非阻塞 I/O：I/O 操作不会阻塞调用线程。如果数据不可用，操作会立即返回，并允许应用程序继续处理其他任务。可以通过轮询或事件驱动的方式来处理数据。

  • 选择器（Selector）：Java NIO 中用于监控多个通道（Channel）上 I/O 事件的对象。选择器可以同时处理多个连接，通过事件驱动机制提高并发性能。

  • 通道（Channel）：Java NIO 中的一个类，用于表示可以进行 I/O 操作的开放连接。与流不同，通道是双向的，可以进行读写操作。
